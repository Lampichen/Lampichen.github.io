[Installationsanleitung]{.underline}

Zur Installation von dem CMS WordPress auf einer VM oder einem Server, sollte bevorzugt Ubuntu, Linux oder Debian als Betriebssystem laufen. Um mit der Einrichtung zu starten, verbindet man sich per SSH mit dem Server. Für die Verbindung sollte man den Root-User oder einen Benutzer mit root-Rechten nutzen. Die Verbindung baut man über folgenden Befehl auf:

ssh benutzer@server_ip

Nach dem erfolgreichen Verbinden mit dem Server, sollte zuerst das System aktualisiert werden. Dies wird gemacht, um sicherzustellen, dass alle Pakete auf dem neusten Stand sind:

sudo apt update

sudo apt upgrade -y

Im nächsten Schritt werden alle benötigten Pakete installiert. Dazu gehören der Webserver „Apache", die Datenbank „MariaDB" sowie PHP und alle für WordPress notwendigen PHP-Erweiterungen. Diese bilden die technische Grundlage für die Anwendung:

sudo apt install apache2 mariadb-server php php-mysql libpache2-mod-php php-cli php-curl php-xml php-mbstring php-zip unzip curl -y

Nachdem diese Installation durchgelaufen ist, kann sowohl der Webserver als auch die Datenbank, bzw. der Datenbankserver gestartet werden. Hierbei aktiveren wir gleich, dass diese beim Systemstart automatischen starten und mitlaufen:

sudo systemctl enable apache2

sudo systemctl start apache2

sudo systemctl enable mariadb

sudo systemctl start mariadb

Nun richten wir die Datenbank für WordPress ein. Dazu melden wir uns zuerst in der Datenbank von Mariadb an. Danach erstellen wir eine neue Datenbank und richten einen neuen Benutzer für WordPress ein. Dieser Nutzer soll zur Sicherheit nur rechte für diese Datenbank erhalten und keine andere.

sudo mysql -u root -p

CREATE DATABASE wordpress DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER \'wpuser\'@\'localhost\'IDENTIFIED BY \'ihr eigenes starkes Passwort\';

GRANT ALL PRIVILEGES ON wordpress.* TO \'wpuser\'@\'localhost\';

FLUSH PRIVILEGES;

EXIT;

Sobald dieser Schritt geschaft ist und die Datenbank damit bereit ist, können wir Anfangen WordPress herunterzuladen und im Webverzeichnis zu installieren. Dafür müssen wir zunächst in das Standard-Webverzeichnis von Apache wechseln:

cd /var/www/html

Danach kann angefangen werden WordPress von der offiziellen Seite herunterzuladen und zu entpacken:

sudo curl -O https://de.wordpress.org/latest-de_DE.tar.gz

sudo tar -xvzf latest-de_DE.tar.gz

sudo mv wordpress/* .

sudo rm -rf wordpress latest-de_DE.tar.gz

Als nächstes müssen wir WordPress konfigurieren. Dafür müssen wir die Beispielkonfigurationsdatei kopieren, öffnen und anschließend bearbeiten, sodass wir die zuvor erstellen Datenbankzugangsdaten eintragen können:

sudo cp wp-config-sample.php wp-config.php

sudo nano wp-config.php

define(\'DB_NAME\', \'wordpress\');\ define(\'DB_USER\', \'wpuser\');\ define(\'DB_PASSWORD\', \'SEHR_STARKES_PASSWORT\');\ define(\'DB_HOST\', \'localhost\');

Damit die Domain korrekt auf die WordPress-Installation zeigt, wird anschließend geprüft ob ein Virtual Host im Apache Webserver erstellt wurde. Dazu öffnet man die Konfigurationsdatei vom Apache Webserver:

sudo nano /etc/apache2/sites-available/000-default.conf

In dieser Datei sollte nun die Domain definiert sein und das WordPress-Verzeichnis als Root-Verzeichnis gesetzt werden. Wenn wie beschrieben WordPress in /var/www/html installiert wurde, muss das Root-Verzeichnis nicht mehr gesetzt werden:

\<VirtualHost *:80>\ ServerName ihre-domain.de\ DocumentRoot /var/www/html\ \ ErrorLog \${APACHE_LOG_DIR}/error.log\ CustomLog \${APACHE_LOG_DIR}/access.log combined\ \</VirtualHost>

Anschließend muss der Apache Server neu geladen werden um gemachte Änderungen zu speichern und zu laden:

sudo systemctl reload apache2

Bereits hier kann man nun versuchen das erstmal mit der Website zu verbinden. Sollte die Wordpress-Anmeldeseite auftauchen, so hat das html-Verzeichnis die bereits benötigten Rechte. Sollte dies jedoch nicht der Fall sein so werden anschließend die Dateirechte angepasst:

sudo chown -R www-data:www-data /var/www/html\ sudo chmod -R 755 /var/www/html

Damit Wordpress von außen erreichbar ist, müssen die benötigten Netzwerkports auf dem Server oder der VM freigegeben werden. Zum einen müssen SSH-Zugriff erlaubt werden, um die Verbindung zum Server/VM nicht zu verlieren von man die Firewall aktiviert. Zudem müssen die Ports 80 und 443 freigegeben werden.

Für Ubuntu Server ist der Befehl:

sudo ufw allow OpenSSH\ sudo ufw allow \'Apache Full\'

sudo ufw enable

Danach kann man den Status der Firewall überprüfen, indem man sudo ufw status eingibt.

Der Eintrag Apache Full bedeutet, dass sowohl Port 80 (HTTP) als auch Port 443 (HTTPS) für eingehende Verbindungen freigegeben sind. Damit Besucher die Webseite über einen Webbrowser erreichen können, muss zusätzlich ein Webserver auf diesen Ports lauschen und gegebenenfalls ein SSL-Zertifikat für HTTPS eingerichtet sein.

[SSL-Zertifikat zur Verschlüsselung]{.underline}

Für die Nutzung von Zertifikaten kann sowohl ein Zertifikat von anderen Unternehmen ausgestellt werden als auch ein selbstsigniertes Zertifikat verwendet werden. Ein SSL Zertifikat ist zur Verschlüsselung der Verbindung da. Selbstsignierte Zertifikate werden bei der Verbindung mit der Website automatisch von Browsern als nicht vertrauenswürdig eingestuft, obwohl sie wie offizielle Zertifikate verschlüsseln. Dementsprechend erscheint beim Verbinden auch eine Sicherheitswarnung.

Um ein selbstsigniertes Zertifikat zu erstellen, nutzen wir OpenSSL. Das erstellt uns einen neuen privaten Schlüssel und ein selbstsigniertes Zertifikat:

sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/webshop.key -out /etc/ssl/certs/webshop.crt

Während der Erstellung werden einige Informationen abgefragt. Besonders wichtig ist der Wert für Common Name (CN). Hier sollte die Domain eingetragen werden, über die die WordPress-Installation erreichbar sein wird, beispielsweise:

Common Name (CN): meine-domain.de

Nach erfolgreicher Erstellung befinden sich die beiden Dateien im Verzeichnis:

/etc/ssl/certs/webshop.crt\ /etc/ssl/private/webshop.key

Der private Schlüssel sollte zudem ausschließlich für Root lesbar sein, weshalb wir diesen nochmal schützen müssen:

sudo chmod 600 /etc/ssl/private/webshop.key

Nun wird das SSL-Modul von Apache aktiviert:

sudo a2enmod ssl\ sudo a2enmod rewrite

Anschließend wird die SSL-Konfiguration angepasst:

sudo nano /etc/apache2/sites-available/default-ssl.conf

Der nachfolgende Teil sollte in der default-ssl.conf vorhanden sein. Wenn teile Fehlen müssen diese hinzugefügt werden:

\<VirtualHost *:443>\ ServerName meine-domain.de\ \ DocumentRoot /var/www/html\ \ SSLEngine on\ SSLCertificateFile /etc/ssl/certs/webshop.crt\ SSLCertificateKeyFile /etc/ssl/private/webshop.key\ \ \<Directory /var/www/html>\ AllowOverride All\ Require all granted\ \</Directory>\ \ ErrorLog \${APACHE_LOG_DIR}/wordpress_ssl_error.log\ CustomLog \${APACHE_LOG_DIR}/wordpress_ssl_access.log combined\ \</VirtualHost>

Danach kann die Datei gespeichert und geschlossen werden. Darauf hin aktivieren wir auch die gerade angepasste Datei:

sudo a2ensite default-ssl.conf

Um nun Besucher automatisch von http auf HTTPS umzuleiten, muss zusätzlich die Konfiguration unter Port 80 angepasst werden:

sudo nano /etc/apache2/sites-available/000-default.conf

\<VirtualHost *:80>\ ServerName meine-domain.de\ \ Redirect permanent / https://meine-domain.de/\ \</VirtualHost>

Danach überprüfen wir die Konfiguration auf ihren Syntax. Dabei sollte die Meldung Syntax OK kommen:

sudo apache2ctl configtest

Anschließend können wir den Webserver neu starten und den Webserver unter https erreichen:

sudo systemctl restart apache2

Für Testumgebungen, interne Netzwerke oder Entwicklungsserver sind selbstsignierte Zertifikate meist ausreichend. Für öffentlich erreichbare Webseiten wird hingegen empfohlen, ein Zertifikat einer offiziellen Zertifizierungsstelle zu verwenden.

[WordPress Einrichtung und Plugins]{.underline}

Sobald die Installation abgeschlossen ist und die Domain korrekt auf den Server zeigt, kann WordPress im Browser aufgerufen werden. Dazu öffnet man die Domain in einem Webbrowser:

https://deine-domain.de

Beim ersten Aufruf erscheint automatisch der WordPress-Installationsassistent. In diesem Schritt werden grundlegende Informationen zur Website festgelegt, darunter der Seitentitel, ein Benutzername für den Administrator, ein sicheres Passwort sowie eine E-Mail-Adresse. Nach Abschluss dieser Einrichtung ist die Installation vollständig abgeschlossen.

Der Login in den Administrationsbereich erfolgt anschließend über folgende URL:

https://deine-domain.de/wp-admin

Hier meldet man sich mit dem zuvor erstellten Admin-Benutzer und Passwort an. Nach erfolgreichem Login gelangt man in das WordPress-Dashboard, also die zentrale Verwaltungsoberfläche der Website.

Plugins erweitern WordPress um zusätzliche Funktionen wie Sicherheit, SEO, Kontaktformulare oder Performance-Optimierungen. Die Installation erfolgt direkt im Dashboard.

Nach dem Login navigiert man im linken Menü zu:

„Plugins → Installieren"

{width="6.268055555555556in" height="3.3493055555555555in"}

Im Suchfeld kann nun der Name eines gewünschten Plugins eingegeben werden, beispielsweise „Wordfence" für Sicherheit oder „Yoast SEO" für Suchmaschinenoptimierung.

Der Ablauf ist immer gleich:

  1. Plugin im Suchfeld auswählen

  2. Auf „Jetzt installieren" klicken

  3. {width="6.228472222222222in" height="3.3652777777777776in"}Nach erfolgreicher Installation auf „Aktivieren" klicken

Sobald ein Plugin aktiviert ist, steht es sofort zur Verfügung und kann je nach Plugin über zusätzliche Menüpunkte oder Einstellungen konfiguriert werden.

[WooCommerce]{.underline}

WooCommerce ist das Hauptplugin, welches sich auf die Einrichtung eines Onlineshops konzentriert. WooCommerce bietet dabei grundsätzlich Funktionen wie Accountverwaltung, Produkte, E-Mail-Sendungen, Gutscheine und Übersichten.

Zuallererst sollte man, bevor man mit dem Customizing von WooCommerce vorfährt in die Einstellungen gehen um Grundlegende Informationen festzulegen.

{width="6.268055555555556in" height="3.2125in"}

Auch unter den Einstellungen von WordPress müssen Informationen angegeben werde, unteranderem, über welche Domain, bzw. Adresse die Website verfügbar sein soll.

{width="6.268055555555556in" height="3.3805555555555555in"}

[Produktverwaltung]{.underline}

Die Verwaltung der Produkte erfolgt über WooCommerce. Über den Menüpunkt „Produkte" unter WooCommerce können neue Artikel angelegt, bestehende Produkte bearbeitet oder nicht mehr benötigte Produkte entfernt werden.

{width="6.268055555555556in" height="3.38125in"}

Beim Anlegen eines Produkts werden zunächst die grundlegenden Informationen wie Produktname, Beschreibung und Preis hinterlegt. Zusätzlich können Produktbilder hochgeladen werden, die später im Shop angezeigt werden. Auch kurze Produktbeschreibungen und Warenkorbbeschreibungen können hierbei hinzugefügt werden. Besonders wichtig ist die Zuordnung von Kategorien. Durch eine saubere Kategorisierung können Kunden Produkte leichter finden und die Navigation innerhalb des Shops wird deutlich verbessert. Darüber hinaus lassen sich Schlagwörter vergeben, die die interne Produktsuche unterstützen.

{width="6.268055555555556in" height="3.3354166666666667in"}

{width="6.268055555555556in" height="3.3618055555555557in"}

Bei Produkten ist es wichtig zu wissen, dass ein einfaches Produkt nur diese eine Art anbietet. Will man also ein Produkt haben was Variable in Größe und/oder Farbe ist, so muss man in dem Feld, wo gerade noch Einfaches Produkt steht, Variables Produkt auswählen.

Um die verschiedenen Varianten nun zu erzeugen, geht man zuerst in den Reiter Eigenschaften.

{width="6.268055555555556in" height="3.357638888888889in"}

Dort kann man nun Eigenschaften hinzufügen. Wichtig ist das man bei der Erstellung von Eigenschaften auch das Häkchen „Für Varianten verwendet" ankreuzt.

{width="6.268055555555556in" height="3.395138888888889in"}

Hat man das Produkt auf Variables Produkt gestellt so wird der Reiter Varianten angezeigt. In diesem können Varianten automatisch aus den Eigenschaften generiert werden. Durch das obere Feld Bulk Actions können dann für alle Varianten eine Eigenschaft ausgewählt bzw. bearbeitet werden, wie z.B. der Preis oder Lagerbestand. Hat man die verschiedenen Varianten und will jede davon selber tracken, z.B. wie viel noch auf Lager ist, so kann man das auch über Bulk Actions einstellen. Dementsprechend fällt dann die Verfolgung des Lagerbestanden unter dem Punkt Lagerbestand weg. Sobald die Lagerverwaltung aktiviert wurde, reduziert WooCommerce die verfügbare Menge automatisch nach jedem Kauf. Dadurch behält der Shop Besitzer jederzeit den Überblick über den verfügbaren Beständen zu jedem Artikel.

{width="6.268055555555556in" height="3.3409722222222222in"}

Außerdem kann über den Reiter Lagerbestand auch eine einheitliche Artikelnummer festgelegt werden. Darüber können dann später die Produkte leichter identifizieren und finden.

{width="6.268055555555556in" height="3.345138888888889in"}

Über die Reiter Allgemein, Versand, Verlinkte Artikel und erweitert kann man individuelle Sachen bzgl. des Produktes festlegen. Nicht nur kann man Mindestalter, Steuerklasse oder auch einen Hersteller angeben, sondern auch festlegen, ob das Produkt nur jeweils einmal pro Bestellung gekauft werden darf oder ob der Shop Besitzer auch Produkte zusätzlich empfehlen möchte.

Für Kategorien, Schlagwörter, Lieferzeiten, Hersteller, Preishinweise und Eigenschaften kann man vorgefertigte Objekte anlegen, sodass diese bei der Produkterstellung ausgewählt werden können.

{width="3.1771423884514434in" height="3.9966666666666666in"}

Um als Beispiel Kategorien zu erstellen, muss man dazu in den Reiter Kategorien unter Produkte. In diesem Punkt kann man einfach und schnell verschiedene Kategorien anlegen. Es können dabei auch Kategorien angelegt werden, welche eine Unterkategorie sind. Das Heißt, es gibt eine Kategorie z.B. namens Oberbekleidung und dann können dazu nochmal Unterkategorien, wie Pullover oder Jacke erstellt werden.

{width="6.268055555555556in" height="3.3361111111111112in"}

Genau dieses Layout ist für all die anderen Unterpunkte ungefähr gleich aufgebaut. Es wird immer ein Name und eine Titelform abgefragt. Häufig wird auch eine Beschreibung angeboten, die man selbstständig für sich tun kann.

[Erweiterte Produktsuche]{.underline}

Um Kunden das Auffinden von Produkten zu erleichtern, wurde das Plugin „Advanced Product Search for WooCommerce" integriert. Während der Eingabe eines Suchbegriffs werden passende Produkte bereits automatisch vorgeschlagen. Dadurch können Besucher schneller zum gewünschten Artikel navigieren, ohne mehrere Kategorien durchsuchen zu müssen. Auch kann über die Suchleiste nach bestimmten Kategorien gefiltert werden, wie z.B. Pullover.

{width="6.268055555555556in" height="3.209722222222222in"}

Im Hintergrund können verschiedene Einstellungen für das Plugin und der Suchfunktion angepasst werden. Unter anderem lässt sich festlegen, wie viele Ergebnisse angezeigt werden und ob Produktbilder oder Kategorien in den Suchvorschlägen erscheinen sollen. Die Einstellungen dafür findet man unter dem Reiter WooCommerce und dann Advanced Product Search.

{width="6.268055555555556in" height="3.388888888888889in"}

Nicht nur können die Suchergebnisse angepasst werden, sondern auch stylische Optionen. Darunter fallen z.B. Optionen wie Farben oder das Aussehen und der Aufbau der Suchleiste.

{width="6.268055555555556in" height="3.3743055555555554in"}

{width="6.268055555555556in" height="3.3784722222222223in"}

[Rabattaktionen und Gutscheine]{.underline}

Für Marketingaktionen stehen die umfangreiche Rabattfunktionen zur Verfügung. Mit den Plugins „Discount Rules for WooCommerce" und „Extended Coupon Features for WooCommerce" können Preisnachlässe und Gutscheinkampagnen erstellt werden. Das Plugin „Extended Coupon Features for WooCommerce" hat dabei keine eigene Übersicht, sondern erweitert die normale Gutscheinfunktion von WooCommerce um weitere Funktionen. Um Einstellungen für die Discount Rules festzulegen, muss man unter dem Reiter WooCommerce nach Discount Rules suchen. Um Gutscheine zu Erzeugen muss man in den Reiter Marketing.

{width="6.268055555555556in" height="3.3847222222222224in"}

Für das Plugin „Discount Rules for WooCommerce" gibt es Standardeinstellungen, wobei man unteranderem einstellen kann, dass die Discount Rules mit den normalen WooCommerce Coupons aktiviert, werden können.

{width="6.268055555555556in" height="3.3875in"}

Die Rabatte können nun beispielsweise abhängig von der Bestellmenge oder dem Warenkorbwert vergeben werden. So kann festgelegt werden, dass Kunden ab drei Produkten einen Preisnachlass von zehn Prozent erhalten oder ab einem bestimmten Einkaufswert versandkostenfrei bestellen können.

So sieht z.B. die Einstellung für einen Rabatt auf alle Artikel aus, welches über die Discount Rules Plugin gemacht wurde. Hier können z.B. Produkte, die Discountart oder auch ein aktiver Zeitraum für den Rabatt ausgewählt und festgelegt werden.

{width="6.268055555555556in" height="3.370138888888889in"}

Zusätzlich können individuelle Gutscheincodes erstellt werden. Diese können zeitlich begrenzt oder auf bestimmte Produkte und Kategorien beschränkt werden. Gutscheine eignen sich besonders für Werbeaktionen oder saisonale Angebote.

{width="6.268055555555556in" height="3.407638888888889in"}

{width="6.268055555555556in" height="3.38125in"}

Beim Erstellen eines Gutscheines kann zum einen durch WooCommerce und durch das extra Plugin ein Gutscheincode eingetragen oder generiert werden. Eine Beschreibung ist dabei optional. Nun kann man hier die Rabattart und auch den Betrag des Gutscheines Festlegen. Zudem kann nur ein Verfallsdatum angelegt und kein Aktivzeitraum festgelegt werden.

Außerdem ist es möglich hierbei Nutzungsbeschränkungen wie Mindestbestellwert, bestimmte Produkte oder Kategorien, für die der Gutschein angelegt wird, festzulegen.

{width="6.268055555555556in" height="3.392361111111111in"}

Zudem gibt es Nutzungsbeschränkungen wie ein Benutzungslimit für alle oder für jeweilige Benutzer. Außerdem ist es möglich, dass man Bedingungen an der Kasse stellt, wie z.B. das zur Einlösung eine bestimmte Versandmethode oder Zahlungsart ausgewählt werden muss oder nur eine bestimmte Nutzerrolle den Gutschein nutzen kann.

{width="6.268055555555556in" height="3.38125in"}

{width="6.268055555555556in" height="3.35625in"}

Unter dem Punkt vermischtes kann man dann festlegen, dass der Gutschein automatisch im Warenkorb eingelöst wird oder nicht.

{width="6.268055555555556in" height="3.5416666666666665in"}

[Bestellungen und Versand]{.underline}

Sobald ein Kunde eine Bestellung aufgibt, erscheint diese automatisch im Reiter WooCommerce und dann Bestellungen. Dort können alle eingegangenen Bestellungen eingesehen und verwaltet werden.

{width="6.268055555555556in" height="3.3715277777777777in"}

Innerhalb einer Bestellung sind sämtliche Kundendaten, gekaufte Produkte sowie Zahlungs- und Versandinformationen hinterlegt. Der Bearbeitungsstatus kann jederzeit angepasst werden. Typische Statuswerte sind „In Wartestellung", „Versendet" oder „Geliefert". Auch können die Versanddetails selbst angepasst werden, indem Sendungsverfolgungsnummern, sowohl an der rechten Seite, als auch bei dem Feld Sendungen eingetragen werden. Unter dem Feld Sendungen kann man nicht nur die Sendungsverfolgungsnummer eintragen, sondern auch den Status der Versendung ändern, zu z.B. „versendet" oder „geliefert".

{width="6.268055555555556in" height="3.3847222222222224in"}

{width="6.268055555555556in" height="3.376388888888889in"}

Für die Versandabwicklung stehen die Erweiterungen „Germanized Shiptastic" sowie die DHL-Anbindung zur Verfügung. Diese stellen Verbindungen zu Versanddienstleister bereit, sodass Tracking zugelassen wird und dementsprechend eine Tracking-/Sendungsverfolgungsnummer angegeben werden kann. Inbegriffene Versanddienstleister in Germanized und den beiden Plugins ist DHL, UPS oder auch Deutsche Post.

Über das Plugin „Erweiterte Sendungsverfolgung für WooCommerce" erhält der Kunde Zugriff auf den aktuellen Versandstatus seiner Bestellung. Die Trackingnummer wird direkt in den E-Mail-Benachrichtigungen angezeigt. Auch kann der Kunde seine Bestellung unter „Mein Konto" und dann Bestellungen verfolgen.

Um Einstellungen direkt am Plugin vorzunehmen, muss man dazu in die Einstellungen des Plugins.

{width="6.268055555555556in" height="3.3520833333333333in"}

Drückt man hier dann auf das Zahnrad kann man sich auch die Trackinginfo, welche in der E-Mail angezeigt wird, anpassen und auf seine Wünsche zuschneiden.

{width="6.268055555555556in" height="3.3118055555555554in"}

Zudem kann man auch verschiedene Versandanbieter hinzufügen, die das Plugin dann unterstützt.

{width="6.268055555555556in" height="3.3652777777777776in"}

[Versandoptionen]{.underline}

Durch Germanized und der Erweiterung durch Shiptastic kann man bereits erste Versanddienstleister hinzufügen lassen, welche man wie im Versandunternehmen bei der Sendungsverfolgung oder als Versandunternehmen anbieten kann.

Um Versandoptionen einzurichten, muss man bei WooCommerce in die Einstellungen gehen und in den Unterpunkt Versand.

{width="6.268055555555556in" height="3.342361111111111in"}

Hier kann man nun neue Versandzonen oder -klassen hinzufügen. Mit einer neuen Versandzone wird eingerichtet, welche Versandarten und Regelungen in den angegebenen Ländern, sowie festlegen, welche Länder in der Versandzone zählen.

{width="6.268055555555556in" height="3.354861111111111in"}

Nachdem man die Versandarten angepasst oder hinzugefügt hat, kann man einstellen, welcher Kartons genutzt werden sollen und unter welchen Bedingungen. Auch kann man allgemeine Einstellungen zu der Versandart vornehmen.

{width="6.268055555555556in" height="3.35in"}

[Rechnungen und Lieferscheine]{.underline}

Das Plugin „PDF Invoices & Packing Slips for WooCommerce" erstellt automatisch Rechnungen und Lieferscheine für eingegangene Bestellungen. Die Dokumente werden als PDF generiert und können sowohl vom Shop Besitzer heruntergeladen als auch automatisch an den Kunden, anbei einer E-Mail, versendet werden oder unter „Mein Konto" und dann Bestellungen abgerufen werden.

{width="6.268055555555556in" height="3.3520833333333333in"}

Das Firmenlogo sowie verschiedene Dokumentinformationen können individuell angepasst werden. Dadurch entsteht ein professionelles Erscheinungsbild gegenüber dem Kunden. Die Einstellungen selbst findet man unter dem Reiter WooCommerce als letzten Punkt.

[Rechtliche Funktionen]{.underline}

Für die Einhaltung der deutschen und europäischen Datenschutz- und E-Commerce-Anforderungen, wurden mehrere rechtliche Erweiterungen installiert.

Das Plugin „Germanized für WooCommerce" ergänzt den Shop um rechtlich notwendige Funktionen wie Lieferzeitangaben, Steuerhinweise, Widerrufsinformationen und Pflichtangaben im Bestellprozess. Auch übernimmt es die Steuerberechnung oder auch rechtlich Relevante Seiten wie AGBs.

{width="6.268055555555556in" height="3.4916666666666667in"}

{width="6.268055555555556in" height="3.3583333333333334in"}

Germanized ist ein sehr vielfältiges Plugin, welches viele Funktionen mit sich bringt. Auch hat es intrigiert, dass SEPA-Lastschriftverfahren und auf Rechnung als Zahlungsoptionen freigeschalten werden. Diese kann man dann in den WooCommerce Einstellungen unter Zahlungen aktivieren und bearbeiten.

[Sicherheit des Shops]{.underline}

Zum Schutz des Webshops wurde Wordfence Security installiert. Die Erweiterung überwacht die Webseite kontinuierlich auf verdächtige Aktivitäten und schützt vor typischen Angriffen auf WordPress-Webseiten.

{width="6.268055555555556in" height="3.3402777777777777in"}

Wordfence verfügt über eine integrierte Firewall, die schädliche Zugriffe blockieren kann. Zusätzlich werden regelmäßige Sicherheitsprüfungen durchgeführt, um veränderte Dateien oder bekannte Schadsoftware frühzeitig zu erkennen. Auch kann man hier Login Security aktivieren wie Captcha oder 2FA. Das dient auch vorranging zum Schutz der Kundendaten und -konten.

{width="6.268055555555556in" height="3.3472222222222223in"}

Im Dashboard von Wordfence können Sicherheitsmeldungen eingesehen und empfohlene Maßnahmen direkt umgesetzt werden.

Sicherheitsaspekte

Thema Sicherheit betrifft jeden Webshop, der öffentlich zugänglich ist. Insbesondere E-Commerce Seiten, die mit Vertrauenswürdigen Daten Arbeiten, brauchen Sicherheit, um Daten wie Kundeninformationen, Adressen oder Kontoinformationen zu schützen.

Ein grundlegender Bestandteil der durchgeführten Sicherheitsmaßnahmen ist die Verwendung eines SSL-Zertifikates. Dadurch wird die Kommunikation zwischen Browser und Nutzer geschützt und verhindert das vertrauliche Daten oder Informationen in Klartext abgefangen werden können.

Zu sehen ist das an der Verbindung, die über HTTPS erfolgt und das durch ein Schlosssymbol in der Adressleiste gekennzeichnet wird.

Auf WordPress kommt, wie in den Funktionen beschrieben, dass Sicherheitsplugin Wordfence für WordPress zum Einsatz. Dieses schützt insbesondere vor automatisierten Brute-Force-Angriffen, bei denen versucht wird, durch wiederholtes Ausprobieren von Passwörtern Zugriff auf den Administrationsbereich zu erhalten. Verdächtige Aktivitäten werden erkannt und entsprechende Zugriffe automatisch blockiert. Zusätzlich verfügt das Plugin über eine integrierte Firewall, die schädliche Anfragen frühzeitig filtert, sowie über Funktionen zur regelmäßigen Überprüfung des Systems auf Malware und manipulierte Dateien. Auch bittet es 2 Faktor Authentifizierung an, um sein Benutzerkonto besser schützen zu können oder Captchas, um Bots fernzuhalten.

Auch auf dem Server/der VM wurden gezielte Sicherheitsmaßnahmen umgesetzt. Die Firewall des Servers/VM ist so konfiguriert, dass ausschließlich notwendige Verbindungen zugelassen werden. Dazu zählen der SSH-Zugang zur VM/zum Server sowie die Ports 80 und 443 für den regulären Webverkehr über HTTP und HTTPS. Alle weiteren Ports bleiben geschlossen, um potenzielle Angriffsflächen zu minimieren. Auch wurde die Weiterleitung von HTTP zu HTTPS konfiguriert, sodass man nur auf geschützten Seiten unterwegs ist.

[API]{.underline}

Die gleich beschriebene API wurde unter zur Hilfenahme von Künstlicher Intelligenz entwickelt. Dabei kam die KI ausschließlich unterstützend bei Fragen oder Problemen zum Einsatz, um eine schnelle Problemlösung zu finden. Sie galt dementsprechend als Inspiration und Wissensquelle.

Das entwickelte Plugin nutzt die WooCommerce Rest API unseres Hauptshops. Dabei werden die Bestelldaten automatisch abgerufen und in das Academy-WordPress übertragen. Ziel dieser Schnittstelle ist es, dass nach dem erfolgten Kauf des XYZ-Academy-Zugangs über den Hauptshop, der Kunde Zugang zur Academy Seite erhält, sodass ein automatischer Onboarding-Prozess entsteht und der Academy Shop Zugänge über andere Shops, verkaufen kann.

Der Zugriff auf die WooCommerce API erfolgt über den Endpoint /wp-json/wc/v3/orders des Hauptshops. Über diesen Endpoint werden die Bestellungen im JSON-Format abgerufen. Der gesamte Aufruf erfolgt als HTTP-GET Request. Sodass der Request erfolgreich ist, wurden der Consumer Key und das Consumer Secret beim Request mitgegeben, um die Anfrage zu authentifizieren. Diese Daten werden über HTTP Basic Authentication im Header des Requestes angegeben.

\'headers\' => [

\'Authorization\' => \'Basic \' . base64_encode(\$xyz_consumer_key . \':\' . \$xyz_consumer_secret)

],

Die erstellen Consumer Key und Secret wurden über die Rest API erst erstellt und mit Leserechten bestückt und danach in unserer API am Anfang eingefügt, sodass wir sie später nur aufrufen brauchen.

Bei der Ausführung des Requests beschränken wir zudem durch einen weiteren Parameter die Anzahl der Bestellungen, die wir zurückbekommen. Hierbei lautet der Parameter „completed", da nur abgeschlossene Bestellungen und Zahlungen für uns von Wichtigkeit sind.

Hierzu sieht der die API-URL wie folgt aus:

https://fbw-lab3.th-brandenburg.de/wp-json/wc/v3/orders?status=completed&per_page=100

Da die Antwort der API wie bereits geschildert als JSON-Format zurück kommt, können wir dieser Antwort Informationen zu den einzelnen Bestellungen entnehmen und/oder prüfen. Beim Auswerten der Informationen durchläuft das Plugin iterativ alle Bestellungen und prüft ob die product_id mit der zuvor festgelegten Produkt-ID übereinstimmen.

\$xyz_academy_product_id = 2616;

Diese ID ist ausschlaggeben, um den Zugang zur Academy zu gewährleisten. Nur Treffer nach der Überprüfung werden weitergenutzt, während die anderen Bestellungen ignoriert werden.

Wurde also der passende Kauf gefunden, werden Kundendaten aus dem JSON-Format extrahiert. Wichtig sind hierbei die E-Mail-Adresse sowie Vor- und Nachname. Diese Daten werden verwendet, um einen neuen Nutzer im System anzulegen, auf den der Kunde dann später zugriefen kann. Wird danach versucht das Kundenkonto anzulegen, so wird zuerst geprüft, ob der Account bereits existiert, sodass nicht doppelte Accounts vorhanden sind.

\$existing_user = get_user_by(\'email\', \$email);

if (\$existing_user) {

if (\$debug) {

echo \'\<p style=\"color:orange;\">User existiert bereits: \' . esc_html(\$email) . \'\</p>\';

}

return \$existing_user->ID;

}

Falls kein Kundenkonto existiert, so wird ein Username, basierend auf der E-Mail erstellt und ein Passwort generiert. Anschließend wird über die WordPress Funktion das Konto erstellt.

\$user_id = wp_create_user(\$username, \$password, \$email);

Im Anschluss daran, wird dem Benutzer eine E-Mail zugesendet, welches die Zugangsdaten zur Academy beinhaltet. Der Versand erfolgt über die interne WordPress Mailfunktion und stellt damit unmittelbar nach dem Kauf sicher, dass der Kunde Zugriff auf die Website hat.

Um eine schnelle und einfache Synchronisation zu gewährleisten haben wir einen Cronjob, welcher im Intervall von 5 Minuten die API-Abfrage durchführt, um ständig auf neue Bestellungen zu lauschen.

add_filter(\'cron_schedules\', function (\$schedules) {

\$schedules[\'every_five_minutes\'] = [

\'interval\' => 300,

\'display\' => \'Alle 5 Minuten\'

];

return \$schedules;

});

Dieser Cronjob wird sowohl beim Aktivieren des Plugins automatisch gestartet als auch beim Deaktivieren automatisch verworfen. Zusätzlich ist im WordPress Dashboard in der Leiste links unten ein Abschnitt für das Plugin, sodass man eigenständig die Synchronisation, also die API-Abfrage ausführen kann.
